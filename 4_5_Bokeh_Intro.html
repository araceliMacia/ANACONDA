Gráficos para la Web con Bokeh¶

La biblioteca Bokeh (http://bokeh.pydata.org/) permite crear visualizaciones en Python, pero orientadas a su utilizacin en navegadores Web. El objetivo es conseguir:

  • visualizaciones interactivas al estilo de D3.js pero permitiendo su programacin en Python
  • y dando soporte a streams de datos.

Dado que un Notebook de Jupyter utiliza HTML para visualizar los resultados, Bokeh se integra con los Notebooks, como puede verse en los ejemplos de bokeh.

aqui cuando mandamos la sentencia en python devuelve un trozo de JS que se ejecuta en el navegador.

Bokeh se inspira en la notacin Grammar of Graphics de Wilkinson, al igual que el paquete ggplot2 de R, pero no son compatibles.

Para instalarlo, se puede utilizar pip o conda. Si tenemos Anaconda lo mejor es:

conda install bokeh

Es fundamental entender que Bokeh es una interfaz que utiliza JavaScript en el navegador para la visualizacin, por lo que la programacin se puede hacer en otros lenguajes que no son Python, por ejemplo en R.

Los tres niveles en Bokeh¶

Bokeh se puede utilizar en tres niveles:

  • low-level bokeh.models interface, mxima flexibilidad, pero hay que hacerlo todo parte a parte.
  • intermediate-level bokeh.plotting interface, centrado en la composicin de glifos visuales (glyphs).
  • high-level bokeh.charts interface que simplifican la tarea de hacer visualizaciones o grficos complejos, tomndolos de un catlogo pre-construido.

from bokeh.charts import Scatter #scatter plot es un diagrama de puntos.
samplesize = 200 # With the full dataset it becomes slow. Esto es para que cargue mas rapido.
tools = "pan, box_zoom, reset, resize" # omit the annoying "wheel_zoom"

p = Scatter(data[:samplesize], x="length", y="rating", 
            color="Short", tools=tools, legend="top_right") # el atributo tool, son las herramientras que se ven alpintalor.
show(p)

Bokeh y Big Data¶

En lo visto hasta aqu, los datos estn en memoria (DataFrame de pandas) y Bokeh los utiliza ntegramente, lo cual deja de ser prctico cuando hay muchos datos a mostrar en un solo grfico.

Bokeh integra la posibilidad de utilizar WebGL para acelerar mediante la Graphics Processing Unit (GPU) el javascript generado. Esto simplemente aprovecha una caracterstica presente en la mayora de los navegadores modernos.

Otra posibilidad es utilizar un Servidor de Bokeh, de modo que los grficos se actualizan en el servidor y los clientes JavaScript proporcionan una vista.

esto es un sw aparte, el programa vive en el servidor y va pasando trozos al navegador ( como si fuera un streaming) de lo que es necesario visualizar, de modo que no se colapsa el navegador.

Por ltimo, Bokeh pertenece al mismo grupo de proyectos que Blaze, una biblioteca que trata de proporcionar una interfaz de programacin parecida a los DataFrames de pandas, pero actuando sobre conjuntos de datos grandes, que en ocasiones no se pueden tener en memoria.
